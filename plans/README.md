## Findings considered and rejected

So nobody re-audits them:

- **CORRECTNESS-01 / CORRECTNESS-04 — "unawaited fire-and-forget Discord alerts
  cause unhandled rejections"**: rejected. The alert chain
  (`sendTaskAdminAlert` → `sendAdminAlert` → `sendDiscordMessage` in
  `src/lib/discord.ts`) wraps its only throwing call (`fetch`) in a try/catch
  that swallows and logs. Not awaiting it is intentional non-blocking telemetry
  and cannot produce an unhandled rejection. No action needed.
- **SEC-04 — "unauthenticated file upload allows malware hosting"**: downgraded,
  not planned as security-critical. `src/app/api/upload/route.ts:7` gates the
  route behind `isUserAuthorizedToEdit({})`, which (per
  `src/lib/auth.ts:62-63`) only returns true for superadmins. The residual item
  is low-priority hardening (add MIME/size limits), not an open attack surface.
- **DEPS-1 — "Prisma adapter/client version mismatch (`@prisma/adapter-pg ^6`
  vs `@prisma/client ^5`) risks runtime errors"**: re-framed and folded into
  plan 002. `@prisma/adapter-pg` is imported nowhere and `driverAdapters` is not
  enabled in `prisma/schema.prisma` (only `postgresqlExtensions`). It is dead
  code, so there is no runtime risk and no need to bump the client to v6 — just
  remove the dependency.
